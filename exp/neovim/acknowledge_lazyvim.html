初步了解lazyvim

注意：以下内容仅为个人初步了解lazyvim的总结，可能存在误解或不完整之处，请谨慎参考。
另外本文并不会太多涉及lazyvim的安装和配置，本文只是帮助大家初步理解lazyvim和vim,以便大家更好地使用nvim。

lazyvim是基于neovim的一个配置框架，它为vim提供了更加简便的配置方式和插件管理方式。
lazyvim之于nvim,就相当于发行版之于linux内核。
安装lazyvim这里不再赘述，可以点击此处进行了解。

首先是目录结构介绍

~/.config/nvim
├── init.lua
├── lua
│   ├── plugins/
│   ├── config/

nvim/config/
├── lazyvim.lua
├── options.lua
├── keymaps.lua

nvim/plugins/
├── example.lua

其中nvim/plugins/里面是我们自定义插件的地方， 当你在plugins里面添加了 插件.lua 后，lazyvim会自动对插件进行识别，安装和配置。 添加插件时建议把某一类插件归为一个 插件类型.lua ，比如要安装AI插件，你想要安装 copilot和 codecompanion, 那么推荐你创建一个 AI.lua, 并在里面写入这两个插件的配置。 

-- plugins/AI.lua
return {
  {
    "github/copilot.vim",
    config = function()
      vim.g.copilot_no_tab_map = true
      vim.api.nvim_set_keymap("i", "", 'copilot#Accept("")', { silent = true, expr = true })
    end,
  },  
  {
    "codecompanion/CodeCompanion.nvim",
    config = function()
      require("codecompanion").setup({
        -- 插件配置项
      })
    end,
  },
}
