DPI / 缩放调校指南

<- 返回 i3 主文档

DPI 影响字体、图标、窗口大小等一切视觉尺度。 如果你发现文字太小/太大、应用缩放不统一、鼠标指针不舒服，先把 DPI 搞清楚，之后再谈美化。

参考文献（想要通透理解的话必读）

  • 先弄清你的屏幕尺寸

    先用 xrandr 看当前分辨率与屏幕名称，之后再决定要不要缩放。

    • xrandr --query

    如果你是 27 寸 2K/4K 屏，默认 DPI 往往不够，字体会显得偏小。

  • 核心方案：用 xrdb 统一 X11 的 DPI（最稳妥）

    xrdb 管理的是 Xresource 数据库；Xft 字体、xterm、urxvt 以及不少 X11 组件都会读取它。 它不会“缩放屏幕”，而是改变字体渲染与 UI 尺度的基准。

    • echo "Xft.dpi: 120" >> ~/.Xresources
    • xrdb -merge ~/.Xresources
    • xrdb -query | rg Xft.dpi

    数值建议从 96 / 110 / 120 / 144 依次试，找到文字与 UI 比例最舒服的点。

    为什么是 xrdb 而不是 xrandr？

    xrandr 的 --scale 是对显示输出做“像素级缩放”，会影响鼠标坐标、截图尺寸、窗口几何；它更像临时硬件缩放方案。

    xrdb 设置的是字体渲染 DPI（Xft.dpi），属于“逻辑尺度”，多数 X11 应用会自动跟随，体验更稳定、更不容易引发连锁问题。

    xrdb 什么时候开始生效？

    它在当前 X 会话里执行后立刻生效；但只对“之后启动的程序”最稳定，已经打开的应用未必会热更新。

    所以你的目标是：让 xrdb 在“i3 启动其他程序之前”就加载完成。

    i3 什么时候开始使用 xrdb？

    i3 本身不会自动读取 Xresources，它只是在启动时读取自己的配置文件。要让 i3 会话中的应用使用 Xresources，必须在 i3 启动流程中手动执行 xrdb。

    简化顺序是：显示管理器启动 X 会话 → i3 读取配置 → i3 依次执行 exec 行 → 你的应用开始启动。

    因此：把 xrdb -merge ~/.Xresources 放在 i3 配置最前面，让它先于 rofi / polybar / 终端等程序执行。

    • # ~/.config/i3/config
    • exec --no-startup-id xrdb -merge ~/.Xresources

    登录管理器要注意：多数 DM（gdm/lightdm/sddm）不会自动读取 ~/.Xresources，所以只写文件是不够的。

    如果你用 startx 或 xinit，建议放在 ~/.xinitrc 顶部，保证应用启动前先加载：

    • # ~/.xinitrc
    • xrdb -merge ~/.Xresources

    如果你是通过 DM 登录，只要把 xrdb 放在 i3 的配置里就够了；它会在 i3 启动后加载并影响之后启动的程序。

  • 可选：让 GTK/Qt 也统一

    如果你常用 GNOME/Qt 应用，而且发现它们没有跟随 Xft.dpi 再考虑这一步。

    • gsettings set org.gnome.desktop.interface scaling-factor 1
    • gsettings set org.gnome.desktop.interface text-scaling-factor 1.2
    • export QT_AUTO_SCREEN_SCALE_FACTOR=0
    • export QT_SCALE_FACTOR=1.2

    为什么是可选？很多 X11 应用只读 Xresources；只有当 GTK/Qt 程序字号异常时，才需要单独覆盖它们的缩放逻辑。

    什么时候需要做？比如：终端和浏览器正常，但文件管理器或设置面板字体明显偏小/偏大。

  • 对单个显示器缩放（xrandr）

    适合双屏且 DPI 不同的情况，单独指定缩放。

    • xrandr --output HDMI-1 --scale 1.25x1.25

    注意 scale 会影响鼠标坐标与截图，能用 DPI 解决优先用 DPI。

  • 常见问题排查

    遇到字体不统一时，先确认以下几件事。

    • fc-cache -f
    • xrdb -query | rg Xft.dpi
    • echo $QT_SCALE_FACTOR

    某些应用会写死缩放，建议优先使用终端或换成支持 DPI 的替代品。
