从窗口管理器到桌面环境🤓

由于orange十分倾向于键盘流，而替代鼠标的一个绝佳工具warpd在使用时需要使用虚拟指针。 但是在 wayland中出于安全考虑，默认不允许应用进行全局输入注入/创建全局虚拟指针，需要合成器与相关协议支持(好事/坏事🤔)。 所以orange反复琢磨之下还是回退到了 X11，并且选择了 i3 作为窗口管理器awa。
因此，本文将从i3出发,以Arch Linux为例，一步步理解和构建一个美观的x11桌面环境。
另外这里也就 不再 过多赘述一些基础的环境配置啦，比如输入法、音量控制、网络管理、字体(nerd系列字体对美化超级有用奥)什么的uwu.

    粉红字可以点的来着 (小声嘀咕)

  • 首先我们需要准备X服务器

    什么是X服务器？它和wayland是什么关系？

    X服务器是一个图形显示服务器，负责管理和显示图形界面。它提供了一个抽象层，使得应用程序可以通过它来绘制图形界面，而不需要直接与硬件交互。
    Wayland是一个新的显示服务器协议，旨在取代X服务器。它提供了更现代化的设计和更好的性能，但由于一些兼容性问题，许多用户仍然选择使用X服务器。

    • sudo pacman -S xorg # 在arch中x全家都被打包到这里面啦

    X11是显示服务器协议(只是个 协议 奥)，Xorg是一个很常见的，根据这个协议开发出来的一个用来管理和绘制图形的服务器( 不要和云服弄混 )。它负责显示与输入转发，是图形栈的关键底层之一🤓。

  • 安装i3窗口管理器

    i3是一个动态平铺式窗口管理器，具有高度的可定制性和键盘友好性。它允许用户通过键盘快捷键来管理窗口布局，提高工作效率。 i3-gaps是i3的一个很好看的分支，它是从原来的仓库分化出来的一个同步更新，但是融入了自己觉得更好看的一些元素。现在i3-gaps已经合并到i3官方仓库了，直接下载i3就好啦uwu。

    • sudo pacman -S i3 # 对于Arch仓库来讲，i3全家(绝大多数基础组件)都在这噢

    i3相当于是一个用来管理应用程序窗口绘制在屏幕上的位置的工具，它并不负责绘制图像奥，它也不负责一些动画之类的东西。

  • 安装feh壁纸工具 #可选，但建议安装

    feh是一款壁纸工具

    • sudo pacman -S feh

    为什么有了i3还要安装一个壁纸工具呢？因为i3本身不负责绘制图像，所以它也不负责设置壁纸奥。feh可以帮助我们设置和管理壁纸，让我们的桌面环境看起来更加美观和个性化uwu.

  • 安装picom合成器 #可选，但建议安装

    picom是一款高性能合成器，它能够为窗口管理器带来美观的动画和窗口效果

    • sudo pacman -S picom

    不要天真地以为窗口管理器会来给你提供特效啦，它只负责布局和管理的(至少i3是这样) :)
    注意，窗口管理器和合成器是不一样的奥，后者通常是负责特效和动画的。

  • 安装polybar #可选，但建议安装

    polybar是一个很美观的状态栏

    • sudo pacman -S polybar

    什么？你说你有i3-status？你不觉得不好看吗？
    快来下载polybar并禁用i3-status :)

  • 安装rofi应用启动器 #可选，但建议安装

    rofi是一个高度可定制的应用程序启动器和窗口切换器，它提供了一个简洁的界面，允许用户快速启动应用程序和切换窗口。

    • sudo pacman -S rofi

    应用启动器，没得说，就是用来快速启动应用的嘛。通常rofi会从 PATH 中寻找可执行文件，并且提供一个界面让用户输入来快速启动应用程序uwu. rofi的启动按键不是固定的，在i3中需要手动绑定按键。这是两种不同的设置方式， 比如在KDE中，rofi的快捷键需自行设置(KDE有个应用启动器叫 Krunner , 你可以用 alt + space 来激活它 )；在i3中，我们可以选择安装快捷键管理工具，也可以直接绑定按键(我目前用的后者owo).

  • 安装autotiling

    autotiling是一个自动平铺工具，它可以根据窗口的数量和布局自动调整窗口的大小和位置，使得桌面环境更加整洁和高效。它可以与i3等窗口管理器配合使用，提供更好的用户体验。

    • sudo pacman -S autotiling

    新装好的i3是不是在处理新窗口的位置和大小时很鸡肋呢，它是不是一个劲地纵向分割屏幕呢，很难看对叭？ 想要自动得到一个美观的平铺布局？ 那就快来试试这个自动贴"瓷砖(窗口)"的工具吧uwu.
